What lives where
lib/data.ts
All const data from the current script block. Typed with interfaces from types.ts. Exported and imported by page components.
lib/stores.ts
Two writable stores: currentVar (selected metric) and fixtureResults (match picks). Persist across route navigations.
routes/+layout.svelte
The sticky nav bar. Uses SvelteKit's $page.url.pathname to highlight the active tab. Zero JS needed for routing.
CSS strategy
Global styles (:root vars, resets, shared classes like .mono, .card) go into app.html or a src/app.css imported in the layout. Component-specific styles (e.g., .group-card) stay in each +page.svelte as <style> blocks — Svelte scopes them automatically, no class-name collisions.
